Stratégie 2 stages

Adaptation rapide → fine-tuning précis · LR décroissant 10×

1
Stage 1 — Adaptation rapide
  • 20 époques · Backbone FROZEN
  • Learning rate : 1×10⁻³
  • Trainable : ~1.05M params (tête seule)
  • Optimizer : Adam + ReduceLROnPlateau
  • Goal : adapter rapidement les features ImageNet
2
Stage 2 — Fine-tuning précis
  • 30 époques · 30 dernières couches UNFROZEN
  • Learning rate : 1×10⁻⁴ (10× plus petit)
  • Trainable : ~15M params
  • EarlyStopping (patience=7) · ModelCheckpoint
  • Goal : affiner les features bas-niveau
Sans 2 stages
⚠ Catastrophic forgetting · convergence ~50 époques
Avec 2 stages
✓ Stable · 30-40 époques · courbes lisses
Durée totale
~30 min GPU · 4-6 h CPU

Grad-CAM via hooks PyTorch

Interprétabilité native · sans dépendance TensorFlow · ~80 ms CPU

def compute_gradcam(model, tensor, target_idx):
    activations, gradients = [], []
    fwd = model.base_model.layer4[-1].register_forward_hook(
        lambda m, i, o: activations.append(o))
    bwd = model.base_model.layer4[-1].register_full_backward_hook(
        lambda m, gi, go: gradients.append(go[0]))

    img = tensor.clone().requires_grad_(True)
    with torch.enable_grad():
        out = model(img)
        one_hot = torch.zeros_like(out)
        one_hot[0][target_idx] = 1.0
        out.backward(gradient=one_hot)
    fwd.remove(); bwd.remove()

    acts, grads = activations[0].squeeze(0), gradients[0].squeeze(0)
    weights = grads.mean(dim=(1, 2))
    cam = (weights[:, None, None] * acts).sum(dim=0)
    cam = F.relu(cam)
    return ((cam - cam.min()) / (cam.max() - cam.min())).cpu().numpy()

Capture les activations de layer4[-1]
📌 Backward hook
Capture les gradients vers la classe cible
📌 Pondération
Moyenne spatiale des gradients → poids par canal
📌 ReLU + normalisation
Heatmap [0,1] → overlay JET sur l'image

Limites actuelles à connaître

Honnêteté scientifique · le projet est éducatif, pas clinique

Un seul dataset Kaggle
Généralisation à d'autres cohortes (BraTS, TCGA-GBM, hôpitaux locaux) non vérifiée. Risque de domain shift.
Pas de cross-validation K-fold
Split unique 70/15/15 stratifié — 98.96% est un point d'estimation, pas un intervalle.
Pas d'ensemble de modèles
Un seul ResNet50. Un ensemble (+ EfficientNet + ViT) gagnerait probablement 0.3-0.7%.
Usage strictement éducatif
Ne remplace pas un radiologue. Pas de certification CE/FDA. Pas de validation clinique.
Pas de grade tumoral
Classification 4-classes uniquement. Pas de distinction Glioma I-II-III-IV ni de segmentation.
Pas de DICOM / EHR
JPEG/PNG uniquement. Pas d'intégration aux systèmes hospitaliers (PACS, HL7, FHIR).
