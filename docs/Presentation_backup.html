Stratégie 2 stages

Adaptation rapide → fine-tuning précis · LR décroissant 10×

1
Stage 1 — Adaptation rapide
  • 20 époques · Backbone FROZEN
  • Learning rate : 1×10⁻³
  • Trainable : ~1.05M params (tête seule)
  • Optimizer : Adam + ReduceLROnPlateau
  • Goal : adapter rapidement les features ImageNet
2
Stage 2 — Fine-tuning précis
  • 30 époques · 30 dernières couches UNFROZEN
  • Learning rate : 1×10⁻⁴ (10× plus petit)
  • Trainable : ~15M params
  • EarlyStopping (patience=7) · ModelCheckpoint
  • Goal : affiner les features bas-niveau
Sans 2 stages
⚠ Catastrophic forgetting · convergence ~50 époques
Avec 2 stages
✓ Stable · 30-40 époques · courbes lisses
Durée totale
~30 min GPU · 4-6 h CPU

Analyse · Anatomie typique des classes

Chaque figure est interprétée — pas juste affichée

📊 fig17 — Image moyenne par classe (200 IRM)
Lecture : moyenne pixel-wise des 200 premières IRM de chaque classe.
👁 Observation clé : les masses de Méningiome et Pituitary sont visibles près de la base du crâne (zone hypophysaire pour la 4ᵉ classe). Glioma a un signal plus diffus → cohérent avec sa nature infiltrante.
🎯 Implication : le modèle a accès à des signatures spatiales distinctes par classe — d'où l'accuracy élevée.
🔍 fig21 — Matrice de similarité cosine inter-classes
Lecture : distance cosine entre images moyennes (1.0 = identique).
👁 Observation clé : Glioma ↔ Méningiome présentent 0.99 de similarité — c'est exactement la paire qui génère 5 erreurs sur 11 dans la matrice de confusion.
🎯 Implication : les erreurs résiduelles sont visuellement justifiées, pas dues à une faiblesse du modèle.

Analyse · Fiabilité des prédictions

Le modèle a-t-il "raison de douter" quand il se trompe ?

📈 fig11 — Confiance : prédictions correctes vs erreurs
Lecture : deux histogrammes superposés (verts = corrects · rouges = erreurs).
👁 Observation clé : les 11 erreurs ont une confiance visiblement plus basse que les 1043 corrects. Le modèle "sait" qu'il hésite.
🎯 Implication : en production, on peut seuiller à 85% et flagger 100% des erreurs pour révision humaine.
🎯 fig16 — Reliability diagram + Brier score
Lecture : bins de confiance vs accuracy réelle (diagonale = parfait).
👁 Observation clé : Brier score ≈ 0.0085 · les barres suivent quasi parfaitement la diagonale → modèle bien calibré.
🎯 Implication : "99% de confiance" signifie vraiment "99% de chances d'être correct" — exploitable pour décisions cliniques.

Analyse · Qualité données & PR

Variabilité du dataset + courbes Precision-Recall

🖼 fig05 — Grille d'échantillons (5 par classe)
Lecture : 5 IRM aléatoires par classe affichées brutes.
👁 Observation clé : grande variabilité d'angles (axial, sagittal, coronal) et de contrastes T1/T2/FLAIR. Pas de pattern visuel évident à l'œil nu pour distinguer Glioma vs Méningiome.
🎯 Implication : le modèle apprend des features non-triviales que l'humain ne voit pas immédiatement.
📐 fig15 — Courbes Precision-Recall + AP
Lecture : precision (Y) vs recall (X) pour chaque classe ; aire = Average Precision.
👁 Observation clé : les 4 courbes sont "plates en haut" avec AP > 0.998. Aucune classe ne souffre d'un trade-off P/R défavorable.
🎯 Implication : le modèle est performant quel que soit le seuil de décision choisi — robuste pour différents contextes cliniques.
